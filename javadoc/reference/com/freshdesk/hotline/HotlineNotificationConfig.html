public HotlineNotificationConfig launchActivityOnFinish (String fullyQualifiedActivityClassName)

Since: API Level 1.0.0

Allows to specify an activity of the app to be launched on up/back naviagation from Hotline Message Activity launched from Notification. Hotline Activity launched from Notification won't have any of the app's activity in the backstack to navigate to. This is helpful to drive engagement back into the application.

Eg. If MainActivity has to be launched when user navigates back/Up from the Hotline Activity, it should be configured as follows

HotlineNotificationConfig config = new HotlineNotificationConfig(); config.launchActivityOnFinish(MainActivity.class.getName()); Hotline.getInstance(context).setNotificationConfig(config);

Parameters
fullyQualifiedActivityClassName Fully qualified name of the activity class. Eg: MyActivity.class.getName();
Returns
  • HotlineNotificationConfig instance to chain config update calls
